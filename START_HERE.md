# Start Here — War Room OS v3.2

## First run

1. Extract the entire ZIP to a normal writable folder.
2. Do not place `.env`, API keys or private tokens inside the repository.
3. Double-click `CHECK_AND_RUN.bat`.
4. Wait for the browser to open.

The header must read:

```text
RESEARCH-FIRST DECISION OS · v3.2
```

## What a healthy start looks like

The dashboard must leave `INITIALIZING`. Depending on credentials and network access,
individual panels may show `LIVE`, `PARTIAL`, `STALE`, `NO_SIGNAL`, `NO_DATA`,
`ACTION_REQUIRED` or `NOT_ENTITLED`. Those are valid explicit states; blank panels and
synthetic replacement data are not.

## Read actions literally

- `TRIGGERED WATCH LONG/SHORT`: entry-state context exists, but it is not an order.
- `WATCH LONG/SHORT`: directional research watch.
- `REDUCE / AVOID`: defensive action; it never creates an IHSG short.
- `NO TRADE`: no permitted action now.
- `CAPITAL_BLOCKED`: evidence has not earned production permission.

## Review order

Open the workspaces in this order:

```text
Mission → Regime & Risk → Opportunities → Markets → Flow & Positioning
→ Causal Map → Execution & Portfolio → Research & Validation
```

At every market, inspect Current Rain, Study the Tape, Counterparty Challenge,
Cost of Staying/Waiting and Next Cheapest Falsifier before interpreting a candidate.
