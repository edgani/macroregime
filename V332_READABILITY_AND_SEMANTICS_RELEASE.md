# War Room OS v3.3.2 — Readability & Semantic Integrity

This release is a clean replacement for v3.3.1. It:

- enlarges only small/secondary text, table rows and Decision Rail details;
- clears recovered worker errors and resets stale runtime state before launch;
- emits one decision row per instrument and resolves contradictory duplicate directions;
- dynamically preserves low-priced crypto precision;
- blocks conflicted and poor-R/R setups from Execution;
- excludes futures, FX, crypto and indices from Company Intel;
- excludes DXY from the pair-specific FX table;
- labels price-relative leadership as context rather than observed capital flow;
- uses CONSTRUCTIVE/ADVERSE for non-trade macro context instead of false LONG/SHORT labels;
- counts production metrics only when their exact grade is PRODUCTION or LIMITED_PRODUCTION;
- keeps Alpha as structural research until value capture and independent proof exist;
- deduplicates provider/status rows; and
- expands dated official-source coverage for current crypto and cross-market developments.

Capital remains blocked. Software correctness, current-source monitoring and descriptive price context do not prove predictive alpha.
